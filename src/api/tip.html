jsmod/ui/fixElement/tip

new require("jsmod/ui/fixElement/tip")(option)

Tip 类依赖 FixElement 类但并未继承，前者多次对后者进行实例化并引用其对象。 既 Tip 为一个或多个目标元素创建相关的 FixElement 对象，而 FixElement 则进行单一的 Dom 元素创建。 两者最主要的区别：实例化 FixElement 时传入参数 target，而实例化 Tip 时传入参数 targets，对 Tip 实例的操作一般都会带上明确的 target 以标识操作哪个 FixElement 对象

className string <optional>
 自定义的 className
title string <optional>
 title 部分的 html, 如果不传则获取 target 上的 data-title 属性, 优先选择 data-title
content string <optional>
 content 部分的 html, 如果不传则获取 target 上的 data-content 属性, 优先选择 data-content
targetType string <optional>
 用 "," 分割可传入一至三个值，每个值都可选 top, right, bottom, left, center
offset Coords <optional>
 tip 的偏移, 如果不传则获取 target 上的 data-offset 属性, 优先选择 data-offset
trigger string <optional>
 hover 触发 tip 的事件, 可选值: hover, click, manual (manual 不为 tip 对象注册任何事状态完全由外部控制)
delay Delay <optional>
 {show:300, hide: 300} 当选择 hover 可用, 设置延迟多少毫秒显示, 消失; 设置为 0 时取消 delay
leavePreventHide bool <optional>
 true 当选择 hover 可用，且设置 delay 时此参数可用，指示是否在鼠标移动到 tip 元素内是否终止隐藏
otherClickHide bool <optional>
 true 当选择 click 可用，当点击页面除 tip 外的 dom 则关闭 tip
preventSelfClickHide bool <optional>
 false 当选择 click 可用，当点击 tip 的当前 target 是否触发隐藏
appendInBody bool <optional>
 false 如果 target 的父元素设置 overflow:hidden 且 relative, absolute 定位时，元素会被隐藏，设置此值为 true 元素将成为 body 的子元素

Example
var Tip = require("jsmod/ui/fixElement/tip");

// 创建实例
new Tip({
    targets: "#target-1, #target-2",
    title: "标题",
    content: "内容"
});
